# 教程一：项目总览与文件结构 —— 从零理解你的个人网站

## 前言：给零基础读者的几句话

如果你从未接触过编程，不用担心。这篇教程假设你**完全零基础**，会用「人话」解释每一个概念。

### 什么是网页？

一个网页本质上就是一个**文本文件**，和 Word 文档没有本质区别。区别在于：
- Word 文档用 `.docx` 结尾，用 Microsoft Word 打开
- 网页用 `.html` 结尾，用**浏览器**（Chrome / Edge / Safari）打开

浏览器的任务就是**阅读 HTML 文件里的文字和指令，然后把它画成你看到的漂亮页面**。

### 构成一个网页的三样东西

每一个现代网页都由三样东西构成，借用盖房子的比喻：

| 技术 | 文件后缀 | 比喻 | 作用 |
|------|----------|------|------|
| **HTML** | `.html` | 房子的**结构**——墙、门、窗的位置 | 决定「页面上有什么」：标题、段落、图片、按钮 |
| **CSS** | `.css` | 房子的**装修**——墙的颜色、窗帘的材质 | 决定「东西长什么样」：颜色、大小、间距、动画 |
| **JavaScript** | `.js` | 房子的**电路和水管**——开关灯、开水龙头 | 决定「东西怎么动」：点击按钮后发生了什么、语言切换、暗色模式 |

**三者缺一不可。** 本项目的所有文件都在这三类之中。

想象你在运营一家餐厅：

| 文件 | 餐厅里的角色 |

| `index.html` | **正门大厅**——客人进来看到的第一眼 |

## 核心概念一：HTML 是怎么工作的

打开 `index.html`，你会看到类似这样的内容：

```html
<!DOCTYPE html>          ← 告诉浏览器「这是最新标准的网页」
<html lang="zh">         ← 网页开始，语言是中文
<head>                   ← 「头部」：给浏览器看的元信息（不显示在页面上）
  <meta charset="UTF-8"> ← 字符编码（让中文不乱码）
  <title>Damara Hu — Personal Website</title> ← 标签页上的标题
</head>
<body>                   ← 「身体」：真正显示在屏幕上的内容
  <h1>你好，我是 Damara Hu</h1>  ← 大标题
  <p>终身学习者</p>              ← 段落
</body>
</html>
```

### 标签（Tag）是什么？

`<h1>你好</h1>` 就是一个**标签**。标签总是成对出现：
- `<h1>` 是**开始标签**（开门）
- `</h1>` 是**结束标签**（关门）
- 中间的文字就是标签的**内容**

常见的标签：

| 标签 | 全称 | 用途 |
|------|------|------|
| `<h1>` ~ `<h6>` | Heading 1~6 | 标题，数字越小字越大 |
| `<p>` | Paragraph | 段落文字 |
| `<a>` | Anchor | 超链接（点击跳转） |
| `<img>` | Image | 图片 |
| `<div>` | Division | 万能容器（装任何东西的盒子） |
| `<button>` | Button | 按钮 |
| `<script>` | Script | 嵌入 JavaScript 代码 |

## 核心概念二：CSS 是怎么工作的

打开 `css/style.css`，你会看到类似这样的规则：

```css
h1 {                          ← 选中所有的 <h1> 标签
  font-size: 2.2rem;          ← 字号：2.2倍基准大小
  color: #1A1A2E;             ← 文字颜色：深灰蓝
  text-align: center;         ← 文字对齐：居中
}
```

CSS 的格式永远是：

```
选择器 {
  属性: 值;
  属性: 值;
}
```

- **选择器**：要装修谁？（`h1` 表示所有大标题）
- **属性**：装修什么？（`color` 表示文字颜色）
- **值**：装成什么样？（`#1A1A2E` 是一种颜色代码）

### 颜色代码 `#1A1A2E` 是什么？

计算机里，颜色用 RGB（红 Red、绿 Green、蓝 Blue）三种光的混合来表示。`#1A1A2E` 是十六进制写法：
- `1A` = 红色分量（26/255，很少红色）
- `1A` = 绿色分量（26/255，很少绿色）
- `2E` = 蓝色分量（46/255，较多蓝色）

结果就是一种**深灰蓝色**。不用死记，用取色器工具选就行。

## 核心概念三：JavaScript 是怎么工作的

打开 `js/i18n-engine.js`：

```javascript
function setLang(lang) {                    // 定义一个叫 setLang 的「功能」
  document.documentElement.lang = lang;     // 把网页语言标记改成新语言
  localStorage.setItem('lang', lang);       // 把选择记住（下次打开还是这个语言）
  updatePageText();                         // 立刻更新页面上所有文字
}
```

### 关键词汇表

| 代码 | 人话 |
|------|------|
| `function` | 「我要定义一个功能」 |
| `setLang` | 这个功能的名字 |
| `(lang)` | 这个功能需要一个输入（切换成什么语言） |
| `{ }` | 功能的具体步骤 |
| `document` | 「当前这个网页」 |
| `localStorage` | 浏览器的「小本子」（可以记东西，关掉浏览器再打开还在） |
| `//` | 注释（给人看的说明，电脑会忽略） |

## 文件之间的依赖关系

打开任何一个 HTML 页面（以 `index.html` 为例），看最底部的三行：

```html
<script src="js/i18n-data.js"></script>    ← ① 先加载翻译字典
<script src="js/i18n-engine.js"></script>  ← ② 再加载翻译引擎（需要用到字典）
<script src="js/shared.js"></script>       ← ③ 最后加载共享脚本（需要用到翻译引擎）
```

**顺序不能乱。** 后面的文件依赖于前面的文件：
- `i18n-engine.js` 需要 `i18n-data.js` 里定义的 `I18N` 变量
- `shared.js` 需要 `i18n-engine.js` 里定义的 `setLang()` 函数

就像盖房子：
1. 先搬砖（i18n-data.js）
2. 再砌墙（i18n-engine.js）
3. 最后装修（shared.js）

## 本项目最重要的设计思想：内容与结构分离

这个网站有一个非常聪明的设计——**把「文字内容」和「代码逻辑」分开了**。

在 `index.html` 里，你不会看到：

```html
<!-- ❌ 不好的做法：把文字直接写死在 HTML 里 -->
<h1>你好，我是 Damara Hu</h1>
```

你看到的是：

```html
<!-- ✅ 好的做法：文字存在数据文件里，HTML 只负责占位 -->
<h1>
  <span data-i18n="home.hero.greeting">你好，我是</span>
  <span data-i18n="home.hero.name">Damara Hu</span>
</h1>
```

`data-i18n="home.hero.greeting"` 是什么？它是一个**钥匙**。翻译引擎拿着这把钥匙去 `js/i18n-data.js` 里查字典，找到对应的中/英文文字，然后替换上去。

### 为什么要这样做？

假设你的网站有 6 个页面、100 处文字。如果不分离：

- 修改一段文字 → 需要在 6 个文件里找到它 → 容易漏改
- 翻译成英文 → 需要把 100 处文字手动翻译 → 极容易出错

分离之后：

- **改文字**：只改 `js/i18n-data.js` 一个文件
- **翻译**：在 `js/i18n-data.js` 里给每段文字加上英文版本
- **改页面结构**：改对应的 HTML 文件

**各管各的，互不干扰。**
