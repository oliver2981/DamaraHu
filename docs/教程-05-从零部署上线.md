# 教程五：从零部署上线 —— 让你的网站被全世界看到

## 前言

前面四篇教程讲完了网站的所有代码。但代码在你自己电脑上，只有你自己能看到。这篇讲**怎么把网站放到互联网上，让任何人在任何地方都能访问**。

---

## 第一部分：基础概念

### 什么是部署（Deploy）？

把你的网站文件从你的电脑传到一台**永远开机的公共服务器**上，这个过程就叫部署。

```
你的电脑（本地）                           Vercel 服务器（云端）
┌──────────────┐                          ┌──────────────────┐
│ index.html   │   ──── 上传 ────→        │ index.html       │
│ style.css    │                          │ style.css        │
│ 各种 JS      │                          │ 各种 JS          │
│ 图片/音乐    │                          │ 图片/音乐        │
└──────────────┘                          └──────────────────┘
  只有你能看                                 全世界都能看
```

### 什么是域名（Domain）？

IP 地址是一串数字（比如 `76.76.21.21`），很难记。域名就是给 IP 地址起了个好记的名字。

```
damarahu.com  →  翻译成  →  76.76.21.21  →  找到那台服务器
```

Vercel 会免费给你一个 `你的项目名.vercel.app` 的域名。你也可以绑定自己的域名。

### 什么是 HTTPS？

你访问网站时，浏览器地址栏前面那个小锁图标 🔒。HTTPS 加密了你和服务器之间的通信，防止中间人偷看。Vercel 自动提供 HTTPS，不需要额外配置。

## 第二部分：准备工作

### 你需要什么

| 东西 | 必须？ | 说明 |
|------|--------|------|
| 一个 GitHub 账号 | ✅ 必须 | 免费注册：[github.com](https://github.com) |
| 一个 Vercel 账号 | ✅ 必须 | 用 GitHub 账号直接登录：[vercel.com](https://vercel.com) |
| Git（版本控制工具） | ✅ 必须 | Windows/Mac 都自带，没有的话去 [git-scm.com](https://git-scm.com) 下载 |
| Vercel CLI | ⚠️ 可选 | 命令行部署工具，`npm i -g vercel` |

### GitHub 是什么？

GitHub 是「代码的云盘」。你把代码上传到 GitHub，然后 Vercel 从 GitHub 读取代码并部署。

Git 和 GitHub 的区别：
- **Git**：工具，跟踪代码的每一次修改（就像 Word 的修订模式）
- **GitHub**：网站，存储 Git 仓库（就像百度网盘存文件）

## 第三部分：第一步 —— 把代码上传到 GitHub

### 步骤 1：在 GitHub 创建新仓库

1. 打开 [github.com](https://github.com)，登录
2. 点右上角 `+` → `New repository`
3. 仓库名填 `my-website`（或其他你喜欢的名字）
4. 选 **Private**（私人仓库）或 **Public**（公开仓库）
5. **不要**勾选「Add a README file」（我们已经有代码了）
6. 点 `Create repository`

### 步骤 2：把本地代码上传

打开终端（Windows 按 Win+R，输入 `cmd`；Mac 打开 Terminal），逐行输入以下命令：

```bash
# ① 进入项目文件夹
cd E:\DamaraHu     # Windows
# cd /Users/你的名字/DamaraHu   # Mac

# ② 初始化 Git（如果还没初始化）
git init

# ③ 添加远程仓库地址（换成你自己创建的仓库地址）
git remote add origin https://github.com/你的用户名/my-website.git

# ④ 添加所有文件到暂存区
git add .

# ⑤ 提交（创建第一个版本记录）
git commit -m "初次提交：个人网站源码"

# ⑥ 推送到 GitHub
git push -u origin master
```

> **如果 ⑥ 报错说分支名不对**，试试 `git push -u origin main`

### Git 的三个区域

```
工作目录（Working Directory）         暂存区（Staging Area）         仓库（Repository）
   你正在编辑的文件          git add →  准备提交的文件    git commit →  已保存的版本
```

- `git add .`：把所有修改加入暂存区（「我要保存这些改动」）
- `git commit`：把暂存区的内容存为一个永久版本（「确认保存」）
- `git push`：把本地仓库的版本同步到 GitHub（「上传到云端」）

## 第四部分：第二步 —— 在 Vercel 部署

### 方法 A：通过网页操作（最简单）

1. 打开 [vercel.com](https://vercel.com)，用 GitHub 账号登录
2. 点 `New Project`
3. 选择你刚上传的仓库 `my-website`
4. Vercel 会自动检测到这是静态网站
5. **不需要改任何配置**，直接点 `Deploy`
6. 等 30 秒 → 部署完成 → 得到一个 `https://xxx.vercel.app` 的网址

**每次你在 GitHub 上推送新代码，Vercel 会自动重新部署。** 完全不需要手动操作。

### 方法 B：通过命令行（适合频繁部署）

```bash
# 安装 Vercel CLI（只需一次）
npm install -g vercel

# 在项目文件夹里登录
vercel login

# 部署
vercel           # 预览部署（测试用）
vercel --prod    # 生产部署（正式上线）
```

## 第五部分：vercel.json 配置文件详解

项目里的 `vercel.json` 告诉 Vercel 怎么部署：

```json
{
  "routes": [
    { "src": "/", "dest": "/index.html" },
    { "src": "/about", "dest": "/about.html" },
    { "src": "/contact", "dest": "/contact.html" }
  ]
}
```

### 这是干什么的？

这叫**路由重写**。正常情况下：

- 访问 `/about.html` ✅ 能找到 about.html 文件
- 访问 `/about` ❌ 没有叫 `about` 的文件，浏览器报 404

有了路由重写：

- 访问 `/about` → Vercel 自动返回 `/about.html` 的内容
- URL 栏还是显示 `/about`（干净、好看）

### 为什么需要这个？

干净的 URL 更好看、更好记、更专业：

| 没有路由重写 | 有路由重写 |
|-------------|-----------|
| `xxx.com/about.html` | `xxx.com/about` |
| `xxx.com/projects.html` | `xxx.com/projects` |
| `xxx.com/contact.html` | `xxx.com/contact` |

## 第六部分：绑定自定义域名（可选）

如果你有自己的域名（比如 `damarahu.com`），可以绑定到 Vercel：

1. 在 Vercel 项目面板 → `Settings` → `Domains`
2. 输入你的域名 → `Add`
3. Vercel 会给你一个 DNS 配置，按照提示去你的域名服务商（阿里云/DNSpod/GoDaddy）添加记录
4. 等几分钟生效 → 你的域名就指向网站了

**Vercel 会自动给你的域名配置 HTTPS 证书（免费），不需要任何额外操作。**

## 第七部分：日常更新流程

每次你想修改网站内容（比如改一段文字、加一张照片、换一张头像），按这个流程操作：

```bash
# ① 修改文件（用任何编辑器改 HTML/CSS/JS 或替换图片）

# ② 看下改了哪些文件
git status

# ③ 把修改加入暂存区
git add .

# ④ 提交
git commit -m "更新了关于我页面的个人介绍"

# ⑤ 推送到 GitHub
git push

# → Vercel 自动检测到推送 → 自动部署 → 30 秒后线上更新
```

你不需要每次都手动运行 `vercel` 命令。GitHub 集成帮你自动化了整个部署流程。

## 第八部分：常见问题排查

### Q: 部署后中文乱码？

检查 HTML 的 `<head>` 里有没有 `<meta charset="UTF-8">`。

### Q: 图片加载不出来？

检查图片路径是**相对路径**（`assets/images/avatar.jpg`）而不是绝对路径（`C:\Users\xxx\avatar.jpg`）。绝对路径在你电脑上能用，但放到服务器上就找不到了。

### Q: 部署后样式丢失？

F12 → Console 看有没有红色的错误。最常见的原因是 CSS 文件路径写错了或 CSS 文件里有语法错误。

### Q: contact.html 密码解密不工作？

Web Crypto API 需要 HTTPS。检查：
1. 网址是 `https://` 开头
2. 不是在本地用 `file://` 打开的
3. 如果本地测试，用 `npx serve .` 启动本地服务器，通过 `http://localhost:3000` 访问

### Q: 怎么回滚到上一个版本？

在 Vercel 项目面板 → `Deployments` → 找到之前的部署 → 点 `...` → `Promote to Production`。秒级回滚。
